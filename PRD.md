# PRD — 미러온 (MirrorON) v3
## 초보자도 3번의 터치로 연결하는 화면 미러링 웹앱

**버전**: 3.0 | **작성일**: 2026-07-04 | **변경 핵심**: 경쟁 서비스(LetsView·scrcpy·삼성 Flow·아이폰 미러링) 분석 기반 "뺄셈 설계" + 원탭 재연결

---

## 1. 제품 개요

### 1.1 한 줄 정의
설치 없이 브라우저만으로 휴대폰 ↔ PC ↔ 태블릿 화면을 양방향 미러링하는 서비스.
**단, 모든 화면은 디지털 초보자가 안내 없이 혼자 따라할 수 있어야 한다.**

### 1.2 제품 철학 — 초보자 퍼스트 (Beginner First) ⭐ v2 핵심
| 원칙 | 의미 |
|---|---|
| **한 화면, 한 행동** | 화면마다 사용자가 해야 할 일은 딱 1가지. 선택지가 2개를 넘지 않음 |
| **큰 글자, 큰 버튼** | 본문 최소 20px, 버튼 글자 24px, 터치 영역 최소 64px. 기본이 크고, "더 크게" 모드 제공 |
| **쉬운 한국어** | "시그널링 연결 중" ❌ → "상대 기기를 찾고 있어요" ✅. 외래어·기술용어 금지 |
| **다음이 항상 보인다** | 모든 단계에 "지금 몇 단계인지(2/3)" + "다음에 뭘 하는지" 표시 |
| **틀려도 괜찮다** | 뒤로가기 항상 가능, 실수해도 데이터 손실 없음, 오류는 해결 방법과 함께 표시 |
| **그림으로 먼저** | 글보다 일러스트/실제 화면 캡처가 먼저. 각 단계에 시각 자료 1개 이상 |

### 1.3 핵심 문제
- 기존 미러링 도구는 설치·설정·용어 장벽으로 초보자가 혼자 사용 불가
- 교육 현장에서 강사가 수강생 기기 세팅에 시간을 뺏김 → **수강생이 스스로 연결**할 수 있어야 함
- 기기 조합(맥북/윈도우 × 갤럭시/아이폰)마다 방법이 달라 혼란

### 1.4 목표
1. **초보자 혼자서 3번의 터치로 연결**: 큰 버튼 클릭 → QR 스캔(또는 6자리 입력) → 승인
2. 4가지 기기 조합 지원 + 조합별 **그림 단계 가이드** 내장
3. 같은 Wi-Fi(P2P) / 다른 네트워크(TURN 릴레이) 모두 동작
4. PWA 홈 화면 추가 + PWABuilder APK — 초보자에게는 "앱 아이콘 하나"로 배포
5. 교육 녹화 모드 (OBS 워크플로우 최적화)

## 2. 경쟁 서비스 분석과 차별화 ⭐ v3 신설

### 2.1 경쟁 서비스가 사용자에게 요구하는 것들

| | LetsView | scrcpy | 삼성 Flow | 아이폰 미러링(Apple) | **미러온** |
|---|---|---|---|---|---|
| 설치 | 폰+PC **양쪽 앱 설치** | PC 설치 + 폰 USB 디버깅(개발자 옵션) | 폰+PC **양쪽 앱 설치** | OS 내장 (macOS 15+, iOS 18+) | **없음 (브라우저)** |
| 계정 | 일부 기능 로그인 | 불필요 | 삼성 계정(일부 기능) | **동일 Apple 계정 + 이중 인증 필수** | **없음** |
| 기기 제한 | 낮음 | Android 전용 | **삼성 폰 전용**, PC는 BT LE/TPM 요건 | **Apple Silicon/T2 맥 + 아이폰 전용, 윈도우 불가** | 낮음 (iOS 송출만 제약) |
| 네트워크 | **같은 Wi-Fi만** | USB 또는 같은 망(ADB 무선) | 블루투스/같은 망 페어링 | 근처 + BT/Wi-Fi | **같은 망 P2P + 다른 망 릴레이 ✅** |
| 연결 절차 | 앱 설치→기기 검색→PIN/QR | 개발자옵션→USB 디버깅→ADB 인증 | BT 페어링→PIN 6자리 상호 확인→기기 등록 | 계정·기기 요건 충족 시 간단 | **큰 버튼→QR→승인 (터치 3번)** |
| 초보자 관점 마찰 | 세션 시간 제한·재연결 요구, 잦은 끊김, 부가기능(화이트보드·녹화 등)으로 복잡한 UI | 사실상 개발자 도구 — 초보자 사용 불가 | 첫 페어링 절차가 길고 요건(TPM 등) 실패 시 원인 파악 어려움 | 요건은 까다로우나 충족 시 매끄러움. 단 생태계 밖(윈도우·갤럭시)은 아예 불가 | — |

### 2.2 차별화 전략 — 뺄셈 설계 (기능을 더하지 않고 뺀다)

경쟁 서비스의 공통 문제는 기능 부족이 아니라 **연결까지의 마찰**이다. 미러온의 차별화는 부가기능 경쟁이 아니라, 초보자가 넘어야 할 문턱을 제거하는 것이다.

**미러온이 사용자에게 요구하지 않는 것 (경쟁사는 요구):**
1. ❌ 설치 — 브라우저 주소 하나 (원하면 PWA/APK로 아이콘화)
2. ❌ 계정·로그인 — 익명으로 전 기능
3. ❌ 같은 제조사/생태계 — 삼성 Flow(갤럭시 전용)·아이폰 미러링(애플 전용)과 달리 조합 무관
4. ❌ 같은 Wi-Fi — LetsView가 못 하는 교차 네트워크를 릴레이로 지원
5. ❌ 개발자 옵션·USB 디버깅·케이블 — scrcpy 대비
6. ❌ 사전 페어링·기기 등록 절차 — 삼성 Flow의 BT 페어링+PIN 상호 확인 단계 제거 (QR 한 번이 곧 페어링)
7. ❌ 부가기능 학습 — 화이트보드·원격제어 등을 초보자 화면에서 제거. 교육 기능은 /admin에 격리

**미러온이 의도적으로 안 하는 것 (Non-Goals 강화):**
- 원격 제어(마우스로 폰 조작) — scrcpy·삼성 Flow의 영역. v2에서도 신중 검토
- 파일 전송·알림 동기화·클립보드 공유 — 미러링 외 기능은 넣지 않는다. "미러링 하나를 제일 쉽게"
- 뷰어 창 안의 도구 팔레트 — LetsView식 플로팅 툴바 금지. 초보자 화면의 버튼은 [멈추기] 하나

### 2.3 경쟁사 대비 신규 반영 기능 (마찰 제거형)
| 기능 | 배경 |
|---|---|
| **[다시 연결] 원탭 재연결** | LetsView의 "시간 제한 후 재연결 요구"가 대표 불만 → 한 번 연결했던 기기 쌍은 다음부터 **버튼 1번**으로 재연결 (기기 기억하기, 30일) |
| **자주 쓰는 PC 고정** | 삼성 Flow의 "기기 등록" 개념을 절차 없이 — 연결 성공 시 "이 컴퓨터를 기억할까요?" 한 번만 물음 |
| **연결 끊김 자동 복구** | LetsView 잦은 끊김 불만 대응 — 재시도 3회 자동, 사용자는 "다시 연결하고 있어요…" 문구만 봄 |
| **요건 사전 점검** | 삼성 Flow처럼 연결 실패 후 원인을 찾게 하지 않음 — 첫 화면 진입 시 브라우저/기능 지원을 미리 감지해 가능한 버튼만 보여줌 |

## 3. 타깃 사용자 & 페르소나

| 페르소나 | 상황 | 핵심 니즈 |
|---|---|---|
| **교육자료 제작자** (관리자 역할) | PC에서 OBS 녹화, 폰 화면 병행 표시 | 지연 1초 미만, 교육 모드, 안정성 |
| **디지털 초보 수강생** (주 페르소나 ⭐) | 강사가 준 QR/코드로 자기 폰 화면을 공유해야 함 | 안 헤매고 혼자 성공. 실패해도 뭘 해야 할지 화면이 알려줌 |
| **시니어 사용자** | 자녀/강사 도움 없이 폰 화면을 TV·PC로 크게 보고 싶음 | 큰 글자, 더 큰 글자 모드, 음성 안내(P2) |
| 점포/현장 지원 | PC 화면을 폰으로 원격 확인 | 역방향 미러링 |

> **설계 검증 기준**: "스마트폰으로 카카오톡만 쓰는 60대 사용자가, 종이 안내문 없이 앱 화면만 보고 5분 안에 연결에 성공하는가?"

## 4. 기기 조합별 지원 매트릭스

### 4.1 폰 화면 → PC/태블릿 (송출: 폰)
| # | 조합 | 지원 방식 | 상태 |
|---|---|---|---|
| 1 | 갤럭시(Android Chrome) → 맥북 | ⚠️ 모바일 브라우저는 화면 캡처 미지원 (2026-07 실측 정정) | 🔶 카메라 모드 + 안내. 화면 송출은 컴패니언 앱(MediaProjection) 로드맵 |
| 2 | 아이폰 → 맥북 | iOS는 브라우저 화면 캡처 미지원 | 🔶 앱이 자동 감지 → 쉬운 말로 안내 + ①카메라 모드 ②그림 가이드(QuickTime) |
| 3 | 갤럭시 → 윈도우 PC | 조합 1과 동일 제약 | 🔶 카메라 모드 + 안내 (조합 1과 동일) |
| 4 | 아이폰 → 윈도우 PC | 조합 2와 동일 제약 | 🔶 카메라 모드 + 그림 가이드 |

> ⚠️ **v3.1 정정**: "폰 브라우저에서 화면 송출"은 iOS·Android 공통으로 플랫폼이 차단함 (caniuse 확인: Android Chrome v149 미지원). PC → 폰 방향(4.2)은 계획대로 완전 동작. LetsView·삼성 Flow가 모두 폰 앱 설치를 요구하는 근본 이유가 이것.

### 4.2 PC/태블릿 → 폰 (송출: PC)
모든 조합 ✅ 완전 지원 (폰은 보기만 하면 되므로 갤럭시·아이폰 모두 가능)

### 4.3 네트워크
| 시나리오 | 방식 | 사용자에게 보여주는 문구 |
|---|---|---|
| 같은 Wi-Fi | P2P 직결 | "빠른 연결이 되었어요 ⚡" |
| 다른 네트워크 | TURN 릴레이 자동 전환 | "연결되었어요 (조금 느릴 수 있어요)" |
| 연결 실패 | 자동 재시도 3회 | "연결이 어려워요. 아래 방법을 해보세요" + 그림 해결 가이드 |

## 5. 핵심 기능 목록

### 5.1 MVP (P0)
| 기능 | 초보자 퍼스트 반영 사항 |
|---|---|
| 첫 화면 | 버튼 딱 2개: **[📱 내 화면 보여주기]** / **[🖥️ 다른 기기 화면 보기]** — 각각 화면 절반 크기 |
| 역할 자동 추천 | 기기 감지 → 폰이면 "보여주기", PC면 "보기"를 크게, 반대는 작게 배치 |
| 룸 연결 | 6자리 코드(숫자만, 3-3 분할 대형 표시) + QR. 초보자는 QR 스캔 우선 유도 |
| 연결 위저드 | 3단계 스텝 표시(1.연결 → 2.승인 → 3.완료), 각 단계 그림 + 한 문장 설명 |
| 권한 승인 도우미 | Android 권한 팝업이 뜨기 **직전에** "곧 이런 창이 떠요. [지금 시작]을 눌러주세요" 미리보기 그림 표시 |
| 화면 송출/뷰어 | WebRTC 기반 (기술 사양은 TRD) |
| **[다시 연결] 원탭 재연결** ⭐v3 | 연결 성공 시 "이 컴퓨터를 기억할까요?" 1회 질문 → 다음 방문 시 첫 화면에 **[🔄 지난번 컴퓨터에 다시 연결]** 대형 버튼 (30일 유지, 터치 1번) |
| 요건 사전 점검 ⭐v3 | 첫 화면 진입 시 기기·브라우저 지원을 감지해 **가능한 버튼만 표시** — 실패 후 원인 찾기 방식(삼성 Flow류) 배제 |
| 글자 크기 모드 | [기본 크게] / [더 크게] 토글 — 첫 화면 우상단 상시 노출 |
| 조합별 그림 가이드 | 4개 조합 × 단계별 실제 화면 캡처 + 큰 글자 설명 |
| 오류 해결 도우미 | 실패 원인별 "이렇게 해보세요" 카드 (기술 용어 없이) |
| PWA + APK | 홈 화면 아이콘 → 앱처럼 실행 |
| 교육 녹화 모드 | 관리자용 (초보자 UI와 분리된 별도 패널) |

### 5.2 P1
- 연결 통계 오버레이(관리자 모드 한정 — 초보자 화면에는 비노출)
- 다중 뷰어(1→4), iOS 카메라 미러 모드, 세션 히스토리

### 5.3 P2
- 음성 안내(TTS) 모드, 원격 포인터, 뷰어 녹화, 다국어

## 6. 사용자 플로우

### 6.1 초보 수강생: 내 폰 화면을 강의장 PC에 보여주기 (터치 3번)
1. 폰 홈 화면의 미러온 아이콘 터치 → 큰 버튼 **[📱 내 화면 보여주기]** 터치 *(터치 1)*
2. "PC 화면에 있는 네모 무늬(QR)를 비춰주세요" → 카메라로 스캔 *(터치 2: 카메라 버튼)*
   - QR이 어려우면 하단 [숫자로 입력하기] — 큰 숫자 키패드
3. 미리보기 그림과 함께 "이제 폰이 물어봐요. **[지금 시작]**을 눌러주세요" → 시스템 팝업 승인 *(터치 3)*
4. ✅ "연결 완료! 이제 PC에서 내 화면이 보여요" + 큰 [멈추기] 버튼 하나만 남김

### 6.2 교육자(관리자): 녹화 준비
1. PC에서 [다른 기기 화면 보기] → 룸 코드/QR 대형 표시 (수강생/본인 폰이 스캔)
2. 연결 후 우측 [교육 모드] → 베젤 프레임·배경·항상 위 창 → OBS 녹화

## 7. 성공 지표 (KPI)

| 지표 | 목표 |
|---|---|
| **초보자 과업 성공률** (도움 없이 첫 연결 성공) | ≥ 90% (사용자 테스트 10인 기준) |
| **첫 연결까지 평균 터치 수** | ≤ 4회 |
| **재연결 터치 수** (기억된 기기 쌍) ⭐v3 | 1회 |
| 연결 성공률 (같은 망 / 교차 망) | ≥ 98% / ≥ 95% |
| 첫 프레임까지 시간 (TTFF) | ≤ 5초 |
| 오류 발생 시 자가 해결률 (해결 카드로 재시도 성공) | ≥ 70% |
| PWABuilder 매니페스트 검증 | 오류 0건 |

## 8. 접근성 요구사항 (필수)
- 글자: 본문 20px↑ / 버튼 24px↑ / 코드 표시 56px↑, "더 크게" 모드는 각 1.25배
- 터치 타겟 64×64px 이상, 버튼 간 간격 16px 이상
