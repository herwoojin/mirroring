## 2. 주차별 상세

### Week 1 — M1 (PROMPT P0, P1)
- 디자인 시스템을 **기능보다 먼저** 구축 — 이후 모든 화면이 BigButton/StepWizard/문구사전 위에서 조립되므로 재작업 방지
- 산출물: /dev-preview 카탈로그, 룸 API

### Week 2~3 — M2 (PROMPT P2) ⭐ 성패 구간
- 3일 내 로컬 탭 2개 미러링 실패 시 원인 분석 집중
- Week 3 갤럭시 실기기: 캡처 권한 플로우 + **PermissionPreview 제스처 체인** 검증
- ⚠️ 리스크: PWA standalone에서 getDisplayMedia 제스처 이슈 → 브라우저 모드 우선 출시 우회

### Week 4 — M3 (TURN)
- 자격증명 API + 자동 폴백. 실측 3개 환경(LTE↔사내망, 카페↔집 등)
- 연결 방식이 사용자에겐 "빠른 연결 ⚡ / 조금 느릴 수 있어요"로만 표현되는지 확인
- ⚠️ 사내망 UDP 차단 → TCP 443 TURN 커버 확인

### Week 5 — M4 (PROMPT P3, P3.5, P4) ⭐ 승부처
- 연결 위저드 3단계 + NumPad + ErrorHelpCard + 4조합 그림 가이드
- **[다시 연결] 원탭 재연결(P3.5)** — LetsView 재연결 불만·삼성 Flow 등록 절차를 이기는 핵심 차별화
- **금요일 1차 사용자 테스트**: 디지털 초보 3인 (가족·동료 섭외)
  - 관찰 기록: 어디서 머뭇거리는지, 어떤 단어를 못 알아듣는지 → 문구/그림 즉시 수정
  - wizard_abandon_step 로그와 관찰 결과 대조
- ⚠️ 리스크: QR 스캔 실패율 높음 → NumPad 경로 접근성을 동급으로 유지했는지 재확인

### Week 6 — M5 (PROMPT P5)
- /admin 교육 모드. **본인이 실제 교육자료 1편 시범 녹화** → 피드백 반영
- 초보자 화면에 관리자 요소가 새어 나오지 않는지 점검

### Week 7 — M6 (PROMPT P6)
- manifest→아이콘→SW→Lighthouse(PWA·A11y 90)→PWABuilder→APK→assetlinks 재배포
- ⚠️ manifest 반복 실패 시 TRD 8.1 필드 단위 대조

### Week 8 — M7 (PROMPT P7)
- 실기기 T1~T9 + 사용자 테스트 U1~U5 전량
- **U2 (60대 5분 내 연결)가 최종 관문** — 실패 시 해당 단계 문구·그림 수정 후 재시도
- KPI 대조(터치 수 ≤4, 자가 해결률 ≥70%), 교육 대상자용 1페이지 안내문 제작
