# GUIDE — 미러온 (MirrorON) v3 구축·운영·사용 가이드

---

## 1. 개발 환경 셋업

### 1.1 사전 준비
- Node.js 20+, Antigravity IDE + Claude Code
- Supabase 프로젝트, metered.ca 계정(TURN 무료 0.5GB/월), Vercel 계정

### 1.2 환경 변수 (.env.local)
```
NEXT_PUBLIC_SUPABASE_URL=https://xxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=...
SUPABASE_SERVICE_ROLE_KEY=...        # 서버 전용
TURN_API_KEY=...
NEXT_PUBLIC_APP_URL=https://mirroron.app
```

### 1.3 Supabase 셋업
1. SQL Editor에서 `0001_init.sql` → `0002_beginner_ux.sql` 순서 실행
2. Realtime Broadcast 활성화 확인, pg_cron 확장 → 만료 룸 청소 잡
3. guide_feedback의 anon INSERT 정책 적용 확인

### 1.4 로컬 실행 & 실기기 테스트
```bash
npm install && npm run dev
```
- getDisplayMedia는 HTTPS 필수 → `npx local-ssl-proxy --source 3443 --target 3000` 또는 Vercel 프리뷰
- 같은 Wi-Fi: "빠른 연결이 되었어요 ⚡" 표시 확인 / LTE 전환: "조금 느릴 수 있어요" 릴레이 전환 확인

## 2. TURN 서버 운영
- 시작: metered.ca 관리형 (API 키 → /api/turn-credentials 프록시)
- 확장: coturn(Azure VM, Korea Central) — `static-auth-secret` HMAC 단기 자격증명,
  포트 3478/udp·tcp + 5349/tcp + 49152-65535/udp, `transport=tcp` 443 병행(사내망 방화벽 대응)
- relay 비율 30% 초과 시 셀프호스팅 전환

## 3. PWA 설치 & PWABuilder APK

### 3.1 폰 설치 (초보자 안내문 그대로 사용 가능)
- **갤럭시**: "인터넷(Chrome)으로 사이트에 들어가면 아래에 '홈 화면에 추가' 안내가 떠요. [추가]를 눌러주세요. 바탕화면에 미러온 아이콘이 생겨요."
- **아이폰**: "사파리 아래 가운데의 ⬆(공유) 버튼 → 아래로 내려 '홈 화면에 추가' → [추가]"

### 3.2 PWABuilder APK 생성 (관리자)
1. 프로덕션 배포 확인 (HTTPS + manifest + SW)
2. pwabuilder.com → URL 입력 → 리포트 Manifest/SW 전 항목 통과 확인
   (단골 누락: id, 512 아이콘, maskable, screenshots)
3. Package for Stores → Android → Package ID `app.mirroron.twa`, 서명키 신규 생성 → **키스토어 백업 필수**
4. zip 안의 assetlinks.json SHA256 → 프로젝트 `public/.well-known/assetlinks.json` 반영 → 재배포
   (누락 시 APK 상단에 주소창 남음)
5. `app-release-signed.apk` 배포 (교육생에게는 Drive 링크 + 설치 그림 안내 1장)

## 4. 사용 시나리오 — 그대로 읽어줄 수 있는 초보자 스크립트 ⭐

### 4.1 수강생: 내 폰 화면을 강의장 PC에 보여주기
> (강사가 PC에 QR을 띄운 상태에서, 아래를 그대로 안내)
1. "휴대폰 바탕화면에서 **미러온** 아이콘을 눌러주세요."
2. "화면 위쪽의 큰 파란 버튼 — **[📱 내 화면 보여주기]** 를 눌러주세요."
3. "카메라가 켜지면, 앞의 PC 화면에 있는 **네모 무늬**를 비춰주세요. 삑 소리가 나면 성공이에요."
   - (QR이 안 되는 분) "아래 **[숫자로 입력하기]** 를 누르고, PC 화면의 **큰 숫자 6개**를 눌러주세요."
4. "'곧 이런 창이 나타나요'라는 그림이 보이면 **[준비됐어요]** 를 누르고, 휴대폰이 물어보는 창에서 **[지금 시작]** 을 눌러주세요."
5. "'연결 완료!'가 나오면 끝이에요. 이제 PC에 여러분 화면이 보여요. 끝낼 때는 빨간 **[멈추기]** 버튼 하나만 누르면 돼요."

### 4.1-1 두 번째부터: [다시 연결] (터치 1번) ⭐v3
> 첫 연결 때 "이 컴퓨터를 기억할까요?"에서 [기억하기]를 눌렀다면:
1. "미러온 아이콘을 누르면, 맨 위에 **[🔄 회사 컴퓨터에 다시 연결]** 버튼이 있어요."
2. "그 버튼 **한 번만** 누르면 연결돼요. PC에 미러온 창이 열려 있어야 해요."

### 4.2 교육자: OBS 녹화 (갤럭시 → 윈도우/맥북 공통)
1. PC 브라우저에서 미러온 → [🖥️ 다른 기기 화면 보기] → 코드/QR 표시
2. 폰으로 4.1 절차 수행 → PC에 폰 화면 수신
3. 우하단 관리자 아이콘 → 교육 모드: 베젤 프레임 + 배경 그린 + [별도 창으로 띄우기]
4. OBS: 디스플레이 캡처(강의 화면) + 윈도우 캡처(미러온 PiP 창) + 크로마키 필터
5. 폰은 충전기 연결 권장, 화면 꺼짐은 wakeLock이 방지

### 4.3 PC 화면 → 폰 (역방향)
- 연결 중 [방향 전환] 또는 처음부터 PC에서 [내 화면 보여주기] → 화면/창/탭 선택 → 폰에서 시청 (아이폰 포함 전 기기 지원)

### 4.4 아이폰 화면을 PC에서 봐야 할 때
- 앱이 자동 안내: "아이폰은 화면을 직접 보내는 기능을 막아두었어요"
- **[카메라로 보여주기]**: 실물·다른 기기 촬영 송출
- **맥북**: 케이블 연결 → QuickTime → 새 동영상 녹화 → 카메라를 iPhone 선택 (가이드 페이지에 그림 단계 내장)
- **윈도우**: 외부 AirPlay 수신 프로그램 안내 (자사 기능 아님을 명시)

## 5. 운영 체크리스트
| 주기 | 항목 |
|---|---|
| 일 | /admin 대시보드: 연결 성공률, 평균 터치 수, 위저드 이탈 단계 |
| 주 | TURN 잔량, **guide_feedback 👎 30% 초과 단계 → 그림/문구 개선** |

## 6. 자주 발생하는 문제
| 증상 | 원인 | 해결 |
|---|---|---|
| 폰에 [내 화면 보여주기]가 없음 | iOS 감지 | 정상 — 카메라 모드/가이드로 자동 안내 |
| 같은 Wi-Fi인데 "느릴 수 있어요" 표시 | 공유기 AP isolation | 공유기 무선 격리 해제 |
| 연결됐는데 검은 화면 | 송출 측 DRM 콘텐츠 | 플랫폼 보호 동작 — 다른 화면 전환 안내 |
| [준비됐어요] 눌렀는데 권한창 안 뜸 | 제스처 체인 끊김(버그) | PROMPT.md 품질 검증 프롬프트로 점검 |
| APK에 주소창 표시 | assetlinks 미반영 | 3.2의 4단계 후 재배포 |
| 몇 분 뒤 끊김 | 배터리 최적화 | Chrome 최적화 제외 (가이드 페이지에 그림 포함) |
| 시니어가 글자 작다고 함 | 기본 모드 | 우상단 [더 크게] 토글 안내 (×1.25) |
| [다시 연결]을 눌렀는데 대기만 함 | 상대 기기(PC)가 미러온을 안 열어둠 | PC에서 미러온 열기 → 자동 연결. 60초 후엔 QR 방식으로 자동 전환됨 |
| [다시 연결] 버튼이 사라짐 | 30일 미사용 만료 또는 브라우저 데이터 삭제 | 정상 — QR로 한 번 연결하면 다시 물어봄 |
